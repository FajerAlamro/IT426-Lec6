Local Search Algorithms

A focused review of local search methods and their applications in optimization

Core Concept

What are Local Search Algorithms?

Heuristic methods for optimization and constraint satisfaction

Local search algorithms are heuristic methods for solving complex optimization and constraint satisfaction problems by iteratively improving a single candidate solution — rather than systematically exploring the entire search space.

The search is "local" because it explores only the immediate neighborhood of the current solution — not all possibilities.

Key Characteristics

  • Operates on a single complete solution at any given time
  • Explores only the immediate neighborhood of the current state
  • Uses very little memory — no need to store the path
  • Well-suited for problems with an astronomically large search space
Real-World Examples

Arranging Furniture in a Room

Everyday optimization under constraints

You want to arrange your bedroom furniture so the room feels open and comfortable. You try a layout, then shift the desk slightly — if it feels better, you keep it. You repeat until no small move improves things.
  • State: the current position of every piece of furniture
  • You only care about the final arrangement, not how you got there
  • Millions of possible layouts — you can't try them all
  • You might settle on a "good enough" layout that isn't truly the best — a local maximum
State = furniture positions Huge search space Path is irrelevant

Planning a Weekly Study Schedule

Balancing multiple competing goals

You need to fit 6 subjects into your week while avoiding clashes with classes, gym, and prayer times — and making sure harder subjects get studied when you're most focused.
  • State: the full weekly schedule with all time slots assigned
  • Multiple conflicting goals: coverage, energy levels, no overlaps
  • You tweak one slot at a time — if the schedule improves, keep the change
  • Only the final schedule matters, not how many drafts you made
Conflicting objectives Iterative improvement

Hill Climbing Algorithm

Hill Climbing Search

Also known as Greedy Local Search

A loop that continually moves in the direction of increasing values (uphill). It picks the first neighbor better than the current state and terminates when no neighbor has a higher value.
  • Modifies the current state to try to improve it
  • Picks the first better neighbor — does not compare all neighbors
  • Terminates at any peak — local OR global maximum

Problems with Hill Climbing

Local Maximum: A peak better than all its neighbors but NOT the global optimum — the algorithm stops and never finds the best solution.
Plateau: A flat area where all neighbors have equal value — no clear direction to move. Basic Hill Climbing stalls here.
Ridge: A sequence of local peaks separated by dips — the agent gets trapped and cannot cross the valley to reach higher ground.
Summary & Comparison

When to Use Local Search?

Good Fit When

  • Only the final solution matters, not the path
  • Search space is huge and cannot be fully explored
  • Memory is limited
  • Multiple conflicting objectives exist

Poor Fit When

  • The path to the solution is important (e.g., navigation)
  • A guaranteed optimal solution is required
  • The search space is small and manageable
